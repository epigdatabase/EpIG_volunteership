# EpIG_volunteership

Repository for volunteers and research assistants of the Epiphyte Inventory Group (EpIG) supporting data mining, cleaning, and standardization of epiphyte datasets.

## About EpIG

The Epiphyte Inventory Group (EpIG) is a global initiative focused on compiling and curating occurrence data for vascular epiphytes. The goal is to support research in biodiversity, biogeography, and conservation through a high-quality, standardized global dataset.

## EpIG Database (EpIG-DB)

EpIG-DB is the main data product of the initiative. It is a continuously evolving database that integrates occurrence records from multiple sources. Each version improves:

- Taxonomic consistency  
- Geographic accuracy  
- Data completeness  
- Standardization (e.g., Darwin Core)  

## Purpose of this repository

This repository is designed to support contributions from volunteers and research assistants working on:

- Data mining from primary and secondary sources  
- Data cleaning (formatting, encoding issues, duplicates)  
- Taxonomic standardization  
- Geographic validation (coordinates and locality data)  
- Harmonization of metadata  

All work conducted here contributes to improving data quality and reproducibility.

## Versioning

All workflows and outputs in this repository are developed for:

> **EpIG-DB Version 2 (v2)**

Data structure:

- `raw/` → original, unmodified datasets  
- `processed/` → cleaned and standardized data  
- `outputs/` → final datasets ready for integration  

Version control ensures full traceability of all modifications.

## Workflow overview

1. Data acquisition / mining  
2. Initial cleaning  
3. Taxonomic standardization  
4. Geographic validation  
5. Metadata harmonization  
6. Export to standardized format  

## Contribution guidelines

- Do not modify raw data directly  
- Follow provided scripts and templates  
- Document all changes clearly  
- Ensure reproducibility of outputs
